Project 1: Maze Generating With Scratch

The first project I am going to show you here is on Scratch.

Here is the link:

  • Scratch Maze Maker - This is a link to Scratch project that randomly generates a maze.

What I Accomplished In This Project

This maze not only generates a maze but also does the following: the maze always has a start and a goal, the maze uses up all the space, the maze is always one block wide and the maze can be expanded easily by editing the variables that controls the size of the maze.

The Flow Of The Algorithm

The maze is created by moving the "check point," initially at the top left corner. The code randomly chooses a direction (up, down, left, or right) and checks if the block two spaces away is a path, out of bounds, or a wall. If it's a wall, the block becomes the new check point. This continues until the check point is "stuck", meaning it can't move in any direction. At this point, the code changes the current check point to a past check point and continues. The process ends when no more paths can be created, ensuring the maze has a start and goal, with all spaces filled. The maze is also expandable by editing the variable that controls the size of maze.

Problems I Faced

While I was writing the code, the code got very long. This happened because there were many selections with many conditions, which led to an even longer code.

How I Solved The Problem

To solve this, I used a function for the first time in my life. By using a function, the length of the overall code got shorter, since I didn't need to have same codes all over the place.

What I Learnt

I learnt that using functions can simplify and shorten code, so it is easier for the author and the reader to read the code. Functions came handy in other projects, so I am glad I learnt how to use it in this project.
